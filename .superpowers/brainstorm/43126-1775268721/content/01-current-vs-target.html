Claude Code UI vs llm-code 現況對比

點擊你最想優先改進的區域

1

Welcome Banner

Claude Code: 簡潔的文字 banner，無 ASCII art，顯示 model/project/directory/permission

llm-code 現況: 大型 ASCII logo + 右側資訊面板（已實作，風格不同）

差異：Claude Code 不用 ASCII art，用更緊湊的文字排版

2

輸入區 + Compact Mode

Claude Code: 固定在底部的輸入框，上方是捲動的對話歷史，類似 chat 介面

llm-code 現況: 傳統 REPL 模式 — 輸入和輸出在同一個滾動流中

差異：最大的架構差異 — Claude Code 是 alternate screen（全螢幕 TUI）

3

Tool 執行顯示

Claude Code: 緊湊的行內顯示 — 工具名+參數在一行，結果在下一行，可展開

llm-code 現況: Unicode box (╭─╮) 包裝，較佔空間

差異：Claude Code 更緊湊，用縮排而非方框

4

Thinking 顯示

Claude Code: 可展開/收合的 thinking block，Ctrl+O 查看，不自動展開

llm-code 現況: 結束後顯示 dim panel（不可互動收合）

差異：Claude Code 的 thinking 預設收合，可切換顯示

5

Status Bar（持久底部列）

Claude Code: 固定底部顯示 model + tokens + cost + mode + shortcuts

llm-code 現況: Ink UI 有簡易 StatusBar，TUI 無固定底部列

差異：TUI 模式完全沒有持久狀態列

6

Permission 互動

Claude Code: 行內顯示 [y/n/a] 選項，單鍵按下立即回應

llm-code 現況: Ink UI 有 dialog，TUI 模式直接跳過（speculative deny）

差異：TUI 的 permission prompt 未實作互動

7

全螢幕 TUI (Alternate Screen)

Claude Code: 使用 alternate screen buffer — 輸入固定在底，對話向上捲動

llm-code 現況: 普通 print 模式，全部在主 buffer 中滾動

差異：這是最根本的架構差異，影響整個 UX
